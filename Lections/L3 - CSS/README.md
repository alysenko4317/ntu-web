# Лекція 3. Основи CSS і оформлення вебсторінок

## Тема

**Каскадні таблиці стилів CSS, оформлення HTML-документів та основи адаптивної вебверстки.**

## Основний напрямок лекції

Лекція присвячена ролі CSS у візуальному оформленні вебсторінок і відокремленні дизайну від HTML-структури. Розглядаються базовий синтаксис CSS, способи підключення стилів, селектори, каскадність, специфічність та успадкування властивостей.

Окрему увагу приділено оформленню тексту й елементів сторінки, адаптивній верстці, використанню Bootstrap і призначенню CSS-препроцесорів для організації складніших таблиць стилів.

## Розглянуті питання

- призначення CSS і відокремлення оформлення від HTML-структури;
- структура CSS-правила;
- підключення зовнішніх, внутрішніх та вбудованих стилів;
- селектори тегів, класів та ідентифікаторів;
- комбіновані селектори й селектори за структурою документа;
- псевдокласи та псевдоелементи;
- каскадність, специфічність і пріоритет стилів;
- успадкування CSS-властивостей;
- кольори, шрифти, розміри та способи відображення елементів;
- адаптивна верстка;
- готові CSS-бібліотеки на прикладі Bootstrap;
- призначення CSS-препроцесорів.

## Демонстраційні приклади

Усі приклади розташовані в каталозі [`src/`](src/). Файли пронумеровані за логікою лекції: від застарілих підходів до оформлення — через базовий CSS — до сучасних фреймворків.

### Способи підключення стилів (0–2)

Перші три приклади показують еволюцію підходів до оформлення HTML — від застарілих тегів до зовнішніх таблиць стилів. Усі три сторінки мають однаковий зміст, але оформлені по-різному.

| Файл | Тема | Що демонструє |
|------|------|---------------|
| [`0.font_tags.html`](src/0.font_tags.html) | Застарілі HTML-теги | Оформлення через `<font>`, `<b>`, `<u>` — так робили до CSS; демонструє, чому цей підхід непрактичний |
| [`1.inline_styles.html`](src/1.inline_styles.html) | Вбудовані стилі | Оформлення через атрибут `style=""` — стилі вже на CSS, але змішані з HTML-розміткою |
| [`2.external_css.html`](src/2.external_css.html) | Зовнішня таблиця стилів | Підключення окремого `.css`-файлу через `<link>` — повне відокремлення оформлення від структури |

Файл `2.external_css.html` за замовчуванням підключає `dark.css`. Змініть значення `href` у тезі `<link>` на одну з альтернативних тем, щоб побачити, як один і той самий HTML виглядає по-різному:

| CSS-файл | Тема оформлення |
|----------|-----------------|
| [`dark.css`](src/dark.css) | Темна тема (білий текст на темному тлі) |
| [`flat.css`](src/flat.css) | Світла плоска тема (синій акцент) |
| [`bw.css`](src/bw.css) | Чорно-біла (мінімалістична) |

### Селектори та властивості CSS (4–6)

| Файл | Тема | Що демонструє |
|------|------|---------------|
| [`4.pseudo.html`](src/4.pseudo.html) + [`pseudo.css`](src/pseudo.css) | Псевдоелементи | `::before` та `::after` — додавання контенту через CSS без зміни HTML |
| [`5.display_none.html`](src/5.display_none.html) | Властивість `display` | Перемикання `display: none` / `display: block` для показу та приховування елементів (з JS) |
| [`6.fixed_absolute.html`](src/6.fixed_absolute.html) | Позиціонування | `position: fixed` (модальне вікно по центру екрана) та `position: absolute` (кнопка закриття у куті вікна) |

### Bootstrap та сучасні елементи (7–a)

| Файл | Тема | Що демонструє |
|------|------|---------------|
| [`7.bootstrap_ui.html`](src/7.bootstrap_ui.html) | Компоненти Bootstrap | Навігаційна панель (`navbar`), адаптивна сітка, картки (`card`) — Bootstrap 4 через CDN |
| [`8.bootstrap_grid.html`](src/8.bootstrap_grid.html) | Сітка Bootstrap | Система `container` → `row` → `col-md-*`; адаптивне розташування колонок на різних екранах |
| [`9.dialog.html`](src/9.dialog.html) | HTML-елемент `<dialog>` | Нативне модальне вікно браузера без бібліотек; методи `showModal()` та `close()` |
| [`a.bootstrap_tabs.html`](src/a.bootstrap_tabs.html) | Вкладки Bootstrap | Компонент tabs з Bootstrap 5; перемикання контенту без перезавантаження сторінки |

### Як переглянути

Усі файли можна відкрити безпосередньо у браузері. Для прикладів з Bootstrap (7, 8, a) потрібне підключення до інтернету (CSS і JS завантажуються з CDN).

> **Зверніть увагу на пару 6 → 9:** обидва приклади вирішують одну задачу — модальне вікно. Файл `6.fixed_absolute.html` реалізує його вручну через CSS-позиціонування і JavaScript, а `9.dialog.html` — через нативний HTML-елемент `<dialog>`. Порівняйте обсяг коду та зручність.

### Що спробувати після лекції

1. Відкрити [`2.external_css.html`](src/2.external_css.html) і по черзі підключити `dark.css`, `flat.css`, `bw.css` — переконатися, що HTML не змінюється, а зовнішній вигляд — кардинально.
2. Додати власне CSS-правило до `pseudo.css` — наприклад, стилізувати `::first-letter` для параграфів.
3. Порівняти [`6.fixed_absolute.html`](src/6.fixed_absolute.html) і [`9.dialog.html`](src/9.dialog.html) — два підходи до модального вікна.
4. Відкрити [`8.bootstrap_grid.html`](src/8.bootstrap_grid.html) і змінити розмір вікна браузера — побачити, як колонки перебудовуються на вузькому екрані.
5. Переглянути `css/style.css` в еталоні ЛР1 ([`Labs/LR1_html_css_basics/`](../../Labs/LR1_html_css_basics/)) як приклад повноцінної таблиці стилів для багатосторінкового сайту.
