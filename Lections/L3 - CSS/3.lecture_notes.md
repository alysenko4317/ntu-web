# Лекція 3. Основи CSS

## 1. Місце CSS у веброзробці

HTML спочатку створювався для структурної розмітки текстових документів. Він дозволяє вказати, що певний фрагмент є заголовком, абзацом, списком, посиланням, таблицею або зображенням.

Згодом вебсторінки перетворилися на повноцінні користувацькі інтерфейси. У браузері з’явилися меню, кнопки, форми, вкладки та діалогові вікна. Вебінтерфейси почали використовуватися також для налаштування маршрутизаторів, серверів, мережевих сховищ та інших пристроїв.

Оформлення безпосередньо в HTML виявилося незручним:

- структура змішувалася з дизайном;
- параметри доводилося повторювати;
- змінювати вигляд багатьох сторінок було складно;
- можливостей HTML не вистачало для складних інтерфейсів;
- було важко підтримувати різні розміри екранів.

CSS — **Cascading Style Sheets**, або каскадні таблиці стилів — відокремлює представлення документа від його структури.

У сучасній веброзробці відповідальність розподіляється так:

- **HTML** — структура і семантика;
- **CSS** — оформлення та компонування;
- **JavaScript** — поведінка й інтерактивність.

## 2. Призначення CSS

CSS дозволяє керувати:

- кольорами;
- шрифтами;
- фоном;
- межами;
- розмірами;
- відступами;
- розташуванням;
- видимістю;
- реакцією на наведення або фокус;
- виглядом сторінки на різних пристроях.

Один CSS-файл може використовуватися на багатьох сторінках. Якщо всі документи підключають `style.css`, зміна одного правила може змінити оформлення всього сайта.

## 3. Синтаксис CSS

```css
p {
    color: #333333;
    font-size: 18px;
}
```

CSS-правило складається з:

- селектора `p`;
- блоку оголошень у фігурних дужках;
- властивостей `color`, `font-size`;
- значень `#333333`, `18px`.

Оголошення записується як:

```css
властивість: значення;
```

Коментар:

```css
/* Коментар у CSS */
```

## 4. Підключення CSS

### 4.1. Зовнішній файл

```html
<link rel="stylesheet" href="css/style.css">
```

Це основний спосіб підключення. Він забезпечує:

- повторне використання;
- відокремлення структури від дизайну;
- зручне редагування;
- кешування браузером;
- єдине оформлення багатьох сторінок.

### 4.2. Внутрішні стилі

```html
<style>
    h1 {
        color: navy;
    }
</style>
```

Підходять для невеликих прикладів або стилів, характерних лише для однієї сторінки.

### 4.3. Вбудовані стилі

```html
<p style="color: red;">Попередження</p>
```

Вбудований стиль застосовується безпосередньо до елемента. Він може бути корисним для швидкої перевірки, але в основному коді ускладнює супровід і каскад.

## 5. Базові властивості

```css
body {
    color: #222;
    background-color: #f5f5f5;
    font-family: Arial, sans-serif;
    font-size: 16px;
    line-height: 1.5;
}
```

Поширені властивості:

- `color` — колір тексту;
- `background-color` — колір фону;
- `font-family` — шрифт;
- `font-size` — розмір шрифту;
- `font-weight` — насиченість;
- `text-align` — вирівнювання тексту;
- `width`, `height` — розміри;
- `border` — межа;
- `padding` — внутрішній відступ;
- `margin` — зовнішній відступ;
- `display` — режим відображення.

## 6. Кольори

### 6.1. Іменовані

```css
color: red;
background-color: lightblue;
```

### 6.2. Шістнадцяткові

```css
color: #ff0000;
```

Формат складається з компонентів червоного, зеленого і синього:

```text
#RRGGBB
```

Приклади:

```css
#000000 /* чорний */
#ffffff /* білий */
#ff0000 /* червоний */
#00ff00 /* зелений */
#0000ff /* синій */
```

### 6.3. RGB і прозорість

```css
color: rgb(255, 0, 0);
background-color: rgba(0, 0, 0, 0.5);
```

Альфа-компонент визначає прозорість від `0` до `1`.

## 7. Селектори

### 7.1. За тегом

```css
p {
    color: #333;
}
```

### 7.2. За класом

HTML:

```html
<p class="warning">Попередження</p>
```

CSS:

```css
.warning {
    color: darkred;
}
```

Клас може повторюватися на багатьох елементах.

### 7.3. Кілька класів

```html
<button class="button button-main">Зберегти</button>
<button class="button button-secondary">Скасувати</button>
```

```css
.button {
    padding: 10px 16px;
    border-radius: 4px;
}

.button-main {
    background-color: royalblue;
    color: white;
}

.button-secondary {
    background-color: lightgray;
}
```

Базовий клас задає спільне оформлення, а додатковий — модифікацію.

### 7.4. За ідентифікатором

```css
#main-panel {
    border: 1px solid #999;
}
```

`id` має бути унікальним. Для повторюваного оформлення краще використовувати класи.

### 7.5. Універсальний селектор

```css
* {
    box-sizing: border-box;
}
```

### 7.6. Групування

```css
h1,
h2,
h3 {
    font-family: Georgia, serif;
}
```

### 7.7. За атрибутом

```css
input[type="email"] {
    border-color: steelblue;
}
```

## 8. Комбінатори

### Нащадок

```css
article p {
    line-height: 1.6;
}
```

Вибирає всі `<p>` усередині `<article>`.

### Безпосередня дитина

```css
nav > ul {
    list-style: none;
}
```

### Наступний сусідній елемент

```css
h2 + p {
    font-weight: bold;
}
```

### Наступні споріднені елементи

```css
h2 ~ p {
    color: #555;
}
```

Надто довгі структурні селектори роблять CSS залежним від точної HTML-структури. У багатьох випадках зрозумілий клас є надійнішим.

## 9. Псевдокласи

Псевдоклас описує стан або положення елемента.

```css
.button:hover {
    background-color: navy;
}

input:focus {
    outline: 2px solid royalblue;
}

a:visited {
    color: purple;
}

li:first-child {
    font-weight: bold;
}

tr:nth-child(even) {
    background-color: #f2f2f2;
}
```

- `:hover` — наведення;
- `:focus` — фокус;
- `:visited` — відвідане посилання;
- `:first-child` — перший дочірній елемент;
- `:nth-child()` — вибір за порядковим номером.

## 10. Псевдоелементи

```css
.note::before {
    content: "Примітка: ";
    font-weight: bold;
}
```

Поширені псевдоелементи:

- `::before`;
- `::after`;
- `::first-line`;
- `::first-letter`;
- `::selection`.

Відмінність:

- клас — `.button`;
- псевдоклас — `.button:hover`;
- псевдоелемент — `.button::before`.

## 11. Каскад

Один елемент може відповідати кільком правилам:

```css
p {
    color: black;
}

.message {
    color: blue;
}

#notice {
    color: red;
}
```

```html
<p id="notice" class="message">Повідомлення</p>
```

Браузер визначає підсумкове значення за каскадом.

На результат впливають:

1. походження стилю;
2. важливість;
3. специфічність;
4. порядок правил.

### 11.1. Стилі браузера

Браузер має власні стилі. Саме тому `<h1>` великий, список має маркери, а посилання — стандартний колір навіть без власного CSS.

### 11.2. Специфічність

У загальному випадку:

- тег має низьку специфічність;
- клас, атрибут і псевдоклас — вищу;
- ідентифікатор — ще вищу;
- вбудований стиль має дуже високий пріоритет.

Якщо два правила мають однакову специфічність, зазвичай перемагає записане пізніше.

### 11.3. `!important`

```css
.warning {
    color: red !important;
}
```

`!important` підвищує пріоритет, але його надмірне використання робить CSS складним. Краще виправити структуру селекторів.

## 12. Успадкування

```css
body {
    color: #333;
    font-family: Arial, sans-serif;
}
```

Вкладений текст зазвичай успадкує колір і шрифт.

Часто успадковуються:

- `color`;
- `font-family`;
- `font-size`;
- `line-height`;
- `text-align`.

Зазвичай не успадковуються:

- `margin`;
- `padding`;
- `border`;
- `width`;
- `height`;
- `display`;
- `position`.

Для конкретної властивості потрібно перевіряти документацію.

Спеціальні значення:

```css
color: inherit;
color: initial;
color: unset;
```

## 13. Властивість `display`

### `block`

Елемент починається з нового рядка й зазвичай займає доступну ширину.

### `inline`

Елемент розміщується всередині текстового рядка.

### `inline-block`

Елемент залишається в рядку, але підтримує зручне задання ширини, висоти та вертикальних відступів.

### `none`

```css
.hidden {
    display: none;
}
```

Елемент не відображається і не займає місця.

JavaScript може додавати або видаляти клас:

```javascript
dialog.classList.toggle("hidden");
```

Так JavaScript керує поведінкою, а CSS — конкретним способом відображення.

## 14. Блокова модель

Кожен елемент складається з:

```text
margin
└── border
    └── padding
        └── content
```

- `content` — вміст;
- `padding` — внутрішній відступ;
- `border` — межа;
- `margin` — зовнішній відступ.

```css
.card {
    width: 300px;
    padding: 20px;
    border: 2px solid black;
    margin: 10px;
}
```

У стандартній моделі `content-box` фактична ширина без `margin`:

```text
300 + 20 + 20 + 2 + 2 = 344 px
```

З урахуванням зовнішніх відступів елемент займає `364 px`.

Через це елементи можуть виходити за межі контейнера або не вирівнюватися.

## 15. `box-sizing`

### `content-box`

`width` задає лише ширину вмісту. `padding` і `border` додаються окремо.

### `border-box`

```css
box-sizing: border-box;
```

`width` уже включає вміст, внутрішні відступи й межу.

Поширене глобальне правило:

```css
*,
*::before,
*::after {
    box-sizing: border-box;
}
```

`margin` у ширину `border-box` не входить.

## 16. Компонування

За замовчуванням елементи розташовуються у нормальному потоці документа.

`position` дозволяє змінити цю поведінку:

```css
.element {
    position: relative;
    top: 10px;
}
```

Поширені значення:

- `static`;
- `relative`;
- `absolute`;
- `fixed`;
- `sticky`.

Історично для колонок використовували `float`, хоча його основне призначення — обтікання зображень текстом.

Сучасні макети переважно створюють через Flexbox і Grid.

### Flexbox

```css
.toolbar {
    display: flex;
    gap: 12px;
    align-items: center;
}
```

### Grid

```css
.layout {
    display: grid;
    grid-template-columns: 240px 1fr;
    gap: 20px;
}
```

## 17. Адаптивний дизайн

Сайт можуть відкрити на моніторі, ноутбуці, планшеті або телефоні. Один фіксований макет не підходить для всіх екранів.

Медіазапит:

```css
.layout {
    display: grid;
    grid-template-columns: 240px 1fr;
}

@media (max-width: 700px) {
    .layout {
        grid-template-columns: 1fr;
    }
}
```

На широкому екрані сторінка має дві колонки, на вузькому — одну.

Адаптивність може передбачати:

- перебудову колонок;
- зміну меню;
- збільшення кнопок;
- зміну порядку блоків;
- скорочення відступів;
- приховування другорядного вмісту.

## 18. CSS-фреймворки

CSS-фреймворки надають готові стилі та компоненти.

Bootstrap містить:

- систему сітки;
- кнопки;
- форми;
- таблиці;
- навігацію;
- модальні вікна;
- адаптивні класи.

```html
<button class="btn btn-primary">Зберегти</button>
```

`btn` задає базовий стиль, `btn-primary` — основний варіант.

Переваги:

- швидка розробка;
- узгоджений вигляд;
- готова адаптивність;
- документація та приклади.

Недоліки:

- шаблонний вигляд;
- залежність від класів фреймворку;
- зайві стилі;
- складніша глибока кастомізація.

Фреймворк не замінює розуміння CSS.

## 19. DevTools

У панелі **Styles** можна побачити:

- усі правила для елемента;
- селектори;
- файл і номер рядка;
- активні та перекреслені властивості;
- успадковані стилі;
- стандартні правила браузера.

Панель **Computed** показує підсумкові значення після каскаду.

DevTools дозволяє:

- тимчасово змінити властивість;
- увімкнути або вимкнути правило;
- додати клас;
- примусово ввімкнути `:hover`;
- переглянути блокову модель.

Зміни у DevTools не редагують вихідний файл і зникають після перезавантаження.

## 20. Поширені помилки

### Змішування оформлення з HTML

Замість:

```html
<p style="color: red;">Помилка</p>
```

краще:

```html
<p class="error">Помилка</p>
```

```css
.error {
    color: red;
}
```

### Надмірне використання `id`

Для повторюваних компонентів краще класи.

### Надто складні селектори

Довгі структурні селектори легко ламаються після зміни HTML.

### Випадкове підбирання відступів

Спочатку потрібно перевірити `display`, блокову модель, Flexbox або Grid.

### Нерозуміння `box-sizing`

`width: 100%` разом із `padding` у режимі `content-box` може зробити елемент ширшим за контейнер.

### Надмірне використання `!important`

Це приховує проблему, але ускладнює подальше перевизначення.

## 21. Підсумки

CSS відокремлює зовнішнє представлення від HTML-структури. Базовою одиницею є правило, що складається із селектора й оголошень.

Якщо до елемента підходить кілька правил, результат визначається каскадом, специфічністю, порядком і успадкуванням.

Для компонування особливо важливо розуміти блокову модель. Фактичний розмір залежить від `content`, `padding`, `border`, `margin` і `box-sizing`.

Для сучасних макетів використовують Flexbox, Grid і медіазапити. Bootstrap та інші фреймворки прискорюють розробку, але не скасовують необхідності знати базовий CSS.

Послідовність налагодження:

```text
вибрати елемент у DevTools
→ перевірити активні правила
→ знайти конфлікт специфічності
→ перевірити успадкування
→ проаналізувати блокову модель
→ виправити вихідний CSS
```
