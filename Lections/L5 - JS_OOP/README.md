# Лекція 5. Об'єктна модель, контекст і модульність JavaScript

## Тема

**Замикання, контекст виконання, прототипи, класи та модульна організація JavaScript-коду.**

## Основний напрямок

Лекція поглиблює знання JavaScript і пояснює механізми, необхідні для створення структурованих клієнтських застосунків. Основну увагу приділено лексичній області видимості, замиканням, визначенню контексту `this`, прототипній моделі успадкування, сучасному синтаксису класів, інкапсуляції та поділу коду на ES-модулі.

## Розглянуті питання

- лексична область видимості та лексичне оточення;
- принцип роботи замикань;
- збереження приватного стану за допомогою замикань;
- особливості `var` і `let` у циклах;
- визначення контексту `this`;
- втрата контексту під час передавання методу;
- використання `call`, `apply` і `bind`;
- відмінності звичайних і стрілочних функцій;
- створення об'єктів через функції-конструктори;
- робота оператора `new`;
- властивість `prototype`;
- прототип об'єкта та прототипний ланцюжок;
- власні й успадковані властивості;
- прототипне успадкування та перевизначення методів;
- оголошення класів за допомогою `class`;
- конструктори, методи, `extends`, `super` і статичні методи;
- гетери, сетери та приватні поля;
- інкапсуляція через класи й замикання;
- властивості об'єктів і їх дескриптори;
- призначення ES-модулів;
- іменований експорт та експорт за замовчуванням;
- імпорт залежностей і область видимості модуля;
- підключення модулів через `type="module"`;
- спливання та перехоплення подій;
- властивості `target` і `currentTarget`;
- делегування подій;
- додавання й видалення обробників через `addEventListener`;
- поділ складнішого клієнтського коду за відповідальністю.

## Демонстраційні приклади

Усі приклади розташовані в каталозі [`src/`](src/). Приклади (1–5) ілюструють ключові концепції лекції.

### Ключові концепції

| Файл / каталог | Тема | Що демонструє |
|----------------|------|---------------|
| [`1.closures.html`](src/1.closures.html) | Замикання | Фабрика лічильників — два незалежних замикання зберігають власний `count`; взаємодія з DOM через `addEventListener` |
| [`2.this_context.html`](src/2.this_context.html) | Контекст `this` | П'ять сценаріїв: виклик як метод, втрата контексту, `bind`, `call`/`apply`, стрілочна функція з лексичним `this` |
| [`3.prototypes_and_classes.html`](src/3.prototypes_and_classes.html) | Прототипи та класи | Один і той самий приклад успадкування (Animal → Dog) реалізований спочатку через функцію-конструктор і `prototype`, а потім через `class`/`extends`; приватні поля, гетери, прототипний ланцюжок |
| [`4.event_delegation.html`](src/4.event_delegation.html) | Делегування подій | Один обробник на `<ul>` замість окремого на кожен `<li>`; `event.target` vs `currentTarget`; динамічне додавання елементів без перереєстрації обробників |
| [`5.es_modules/`](src/5.es_modules/) | ES-модулі | `index.html` з `<script type="module">`; `main.js` імпортує іменований експорт із `math.js` та `export default` із `greeter.js` |

### Як переглянути

Більшість прикладів (1–4) можна відкрити безпосередньо у браузері. Для прикладів, що використовують ES-модулі або AJAX, потрібен веб-сервер:

```
cd "Lections/L5 - JS_OOP/src/5.es_modules"
python -m http.server 8080
```

Потім відкрити **http://localhost:8080** у браузері.

### Що спробувати після лекції

1. Відкрити [`1.closures.html`](src/1.closures.html) і додати кнопку «Скинути», яка обнуляє лічильник (підказка: повернути з `createCounter` об'єкт з двома методами).
2. У [`2.this_context.html`](src/2.this_context.html) створити об'єкт із методом і передати його як callback у `setTimeout` — побачити втрату контексту і виправити через `bind`.
3. У [`3.prototypes_and_classes.html`](src/3.prototypes_and_classes.html) додати клас `Cat` із власним `speak()` та перевірити `instanceof`.
4. У [`4.event_delegation.html`](src/4.event_delegation.html) додати кнопку «Видалити останній» — переконатися, що делегування працює без змін.
5. Розширити [`5.es_modules/`](src/5.es_modules/) — додати новий модуль `formatter.js` і підключити його до `main.js`.
6. Пройти квіз у [`_5. closures_and_variables/`](src/_5.%20closures_and_variables%20(quiz_2)/) — спрогнозувати результат до відкриття в браузері.
