# Лекція 5. Об’єктна модель, контекст і модульність JavaScript

## 1. Особливості об’єктної моделі

У класичних ООП-мовах спочатку оголошують клас, а потім створюють його екземпляри. У JavaScript об’єкт можна створити одразу:

```javascript
const user = {
    name: "Anna",
    showName() {
        console.log(this.name);
    }
};
```

Властивості можна додавати під час виконання:

```javascript
user.age = 20;
```

Історично успадкування JavaScript базується на прототипах. Синтаксис `class` з’явився пізніше і є зручнішою формою запису тієї самої моделі.

## 2. Лексична область видимості

```javascript
const globalValue = 1;

function outer() {
    const outerValue = 2;

    function inner() {
        const innerValue = 3;
        console.log(globalValue, outerValue, innerValue);
    }

    inner();
}
```

Внутрішня функція бачить власні змінні, змінні зовнішньої функції та глобальні змінні.

Область називається лексичною, тому що визначається місцем написання функції, а не місцем її подальшого виклику.

## 3. Замикання

```javascript
function createCounter() {
    let value = 0;

    return function () {
        value += 1;
        return value;
    };
}

const counter = createCounter();

counter(); // 1
counter(); // 2
```

Після завершення `createCounter()` змінна `value` продовжує існувати, оскільки повернена функція має доступ до її лексичного оточення.

Замикання зберігає доступ до змінної, а не просто копію її значення.

### Приватний стан

```javascript
function createAccount(initialBalance) {
    let balance = initialBalance;

    return {
        deposit(amount) {
            if (amount > 0) {
                balance += amount;
            }
        },

        getBalance() {
            return balance;
        }
    };
}
```

Зовнішній код не має прямого доступу до `balance`.

## 4. Замикання в циклах

```javascript
const functions = [];

for (var i = 0; i < 3; i += 1) {
    functions.push(function () {
        console.log(i);
    });
}
```

Усі функції виведуть `3`, тому що вони замикають одну змінну `i`.

З `let` для кожної ітерації створюється окреме зв’язування:

```javascript
for (let i = 0; i < 3; i += 1) {
    functions.push(() => console.log(i));
}
```

Результат: `0`, `1`, `2`.

## 5. Контекст `this`

У звичайній функції `this` визначається способом виклику.

### Виклик як методу

```javascript
const user = {
    name: "Anna",
    showName() {
        console.log(this.name);
    }
};

user.showName();
```

Тут `this` дорівнює `user`.

### Втрата контексту

```javascript
const show = user.showName;
show();
```

Функція більше не викликається як властивість `user`. У строгому режимі `this` буде `undefined`.

Посилання на функцію саме по собі не зберігає об’єкт-контекст.

## 6. `call`, `apply`, `bind`

```javascript
function introduce(greeting) {
    console.log(`${greeting}, я ${this.name}`);
}
```

`call` викликає одразу:

```javascript
introduce.call(user, "Вітаю");
```

`apply` передає аргументи масивом:

```javascript
introduce.apply(user, ["Вітаю"]);
```

`bind` повертає нову функцію:

```javascript
const bound = introduce.bind(user);
bound("Вітаю");
```

`bind` корисний, коли метод передається як callback.

## 7. Стрілочні функції та `this`

Стрілочна функція не має власного `this`, а бере його з зовнішнього контексту:

```javascript
const object = {
    value: 10,

    delayedShow() {
        setTimeout(() => {
            console.log(this.value);
        }, 100);
    }
};
```

Стрілка зберігає `this` методу `delayedShow`.

Але стрілка як метод об’єкта часто є помилкою:

```javascript
const object = {
    value: 10,
    show: () => console.log(this.value)
};
```

`this` тут не буде самим `object`.

Стрілочну функцію не можна використовувати через `new`.

## 8. Функції-конструктори

```javascript
function Person(name, age) {
    this.name = name;
    this.age = age;
}

const person = new Person("Anna", 20);
```

Під час `new`:

1. створюється новий об’єкт;
2. його прототип пов’язується з `Person.prototype`;
3. конструктор викликається з новим об’єктом як `this`;
4. повертається створений екземпляр.

Назву конструктора зазвичай починають з великої літери.

## 9. Методи й дублювання

Невдалий варіант:

```javascript
function Person(name) {
    this.name = name;

    this.showName = function () {
        console.log(this.name);
    };
}
```

Кожен екземпляр отримує окремий об’єкт функції.

Краще:

```javascript
function Person(name) {
    this.name = name;
}

Person.prototype.showName = function () {
    console.log(this.name);
};
```

Метод один і використовується всіма екземплярами.

## 10. Прототипний ланцюжок

Кожен звичайний об’єкт має внутрішнє посилання `[[Prototype]]`.

```javascript
Object.getPrototypeOf(person);
```

Під час звернення до властивості JavaScript шукає її:

1. у самому об’єкті;
2. у його прототипі;
3. у прототипі прототипу;
4. до `null`.

```text
person
→ Person.prototype
→ Object.prototype
→ null
```

Не слід плутати:

- `Person.prototype` — властивість функції;
- `Object.getPrototypeOf(person)` — прототип конкретного об’єкта.

## 11. Власні й успадковані властивості

```javascript
Object.hasOwn(person, "name");     // true
Object.hasOwn(person, "showName"); // false
```

Якщо в самому об’єкті створити властивість з тим самим ім’ям, вона перекриє прототипну.

## 12. Прототипне успадкування

```javascript
function Person(name) {
    this.name = name;
}

Person.prototype.showInfo = function () {
    console.log(this.name);
};

function Student(name, group) {
    Person.call(this, name);
    this.group = group;
}
```

`Person.call` ініціалізує поля, але не зв’язує прототипи.

```javascript
Student.prototype = Object.create(Person.prototype);
Student.prototype.constructor = Student;
```

Перевизначення:

```javascript
Student.prototype.showInfo = function () {
    Person.prototype.showInfo.call(this);
    console.log(this.group);
};
```

## 13. Класи

```javascript
class Person {
    constructor(name) {
        this.name = name;
    }

    showInfo() {
        console.log(this.name);
    }
}

class Student extends Person {
    constructor(name, group) {
        super(name);
        this.group = group;
    }

    showInfo() {
        super.showInfo();
        console.log(this.group);
    }
}
```

Методи класу розміщуються в прототипі.

`extends` налаштовує прототипний зв’язок, а `super` викликає базовий конструктор або метод.

Класи є синтаксичною надбудовою над прототипами.

## 14. Статичні методи

```javascript
class Person {
    static fromObject(data) {
        return new Person(data.name);
    }
}
```

Виклик:

```javascript
Person.fromObject({ name: "Anna" });
```

Статичний метод належить класу, а не екземпляру.

## 15. Гетери й сетери

```javascript
class Rectangle {
    constructor(width, height) {
        this.width = width;
        this.height = height;
    }

    get area() {
        return this.width * this.height;
    }

    set size(value) {
        if (value <= 0) {
            throw new Error("Розмір має бути додатним");
        }

        this.width = value;
        this.height = value;
    }
}
```

Використання:

```javascript
rectangle.area;
rectangle.size = 10;
```

## 16. Приватні поля

```javascript
class Account {
    #balance = 0;

    deposit(amount) {
        if (amount > 0) {
            this.#balance += amount;
        }
    }

    get balance() {
        return this.#balance;
    }
}
```

Зовнішній доступ до `#balance` є синтаксичною помилкою.

Назва `_balance` є лише домовленістю і не створює справжньої приватності.

## 17. Замикання чи клас

Фабрика із замиканням:

```javascript
function createCounter() {
    let value = 0;

    return {
        increment() {
            value += 1;
        },

        get value() {
            return value;
        }
    };
}
```

Клас:

```javascript
class Counter {
    #value = 0;

    increment() {
        this.#value += 1;
    }

    get value() {
        return this.#value;
    }
}
```

Замикання зручне для функціонального стилю й невеликих фабрик. Клас — для багатьох екземплярів, спільних методів та успадкування.

## 18. Дескриптори властивостей

```javascript
Object.defineProperty(user, "id", {
    value: 100,
    writable: false,
    enumerable: true,
    configurable: false
});
```

- `writable` — чи можна змінювати;
- `enumerable` — чи видно під час перерахування;
- `configurable` — чи можна видалити або перевизначити.

```javascript
Object.getOwnPropertyDescriptor(user, "id");
```

## 19. Навіщо потрібні модулі

Глобальні змінні створюють:

- конфлікти імен;
- неявні залежності;
- залежність від порядку завантаження;
- складність тестування;
- випадковий доступ до внутрішньої реалізації.

ES-модуль має власну область видимості й відкриває лише явно експортовані значення.

## 20. Іменований експорт

`math.js`:

```javascript
export function add(a, b) {
    return a + b;
}

export const PI = 3.14159;
```

`main.js`:

```javascript
import { add, PI } from "./math.js";
```

Перейменування:

```javascript
import { add as sum } from "./math.js";
```

Простір імен:

```javascript
import * as math from "./math.js";
```

## 21. Експорт за замовчуванням

```javascript
export default class UserService {
    loadUsers() {
        // ...
    }
}
```

```javascript
import UserService from "./user-service.js";
```

Модуль може мати один `default`-експорт і багато іменованих.

## 22. Підключення модуля

```html
<script type="module" src="js/main.js"></script>
```

Модулі:

- мають власну область видимості;
- автоматично працюють у строгому режимі;
- підтримують `import` і `export`;
- завантажуються відкладено;
- підпорядковуються політиці походження.

Їх слід запускати через HTTP, а не через `file://`.

## 23. Динамічний імпорт

```javascript
const module = await import("./chart.js");
module.showChart();
```

`import()` повертає Promise і дозволяє завантажити код лише за потреби.

## 24. Події: `target` і `currentTarget`

```javascript
const menu = document.querySelector(".menu");

menu.addEventListener("click", event => {
    console.log(event.target);
    console.log(event.currentTarget);
});
```

- `target` — елемент, на якому подія виникла;
- `currentTarget` — елемент, чий обробник виконується.

## 25. Спливання

Якщо клікнути вкладену кнопку, подія проходить від неї до предків.

```javascript
panel.addEventListener("click", event => {
    console.log(event.target);
});
```

Зупинка:

```javascript
event.stopPropagation();
```

Її слід використовувати обережно.

Скасування стандартної дії:

```javascript
event.preventDefault();
```

## 26. Делегування подій

```javascript
const list = document.querySelector(".user-list");

list.addEventListener("click", event => {
    const button = event.target.closest("[data-action='remove']");

    if (!button || !list.contains(button)) {
        return;
    }

    button.closest("li").remove();
});
```

Переваги:

- один обробник замість багатьох;
- підтримка динамічно доданих елементів;
- централізована логіка.

## 27. Додавання й видалення обробників

```javascript
function handleClick(event) {
    console.log(event.target);
}

button.addEventListener("click", handleClick);
button.removeEventListener("click", handleClick);
```

Потрібно передати те саме посилання на функцію. Нова анонімна функція з таким самим текстом є іншим об’єктом.

## 28. Приклад розкривного меню

```javascript
const menu = document.querySelector(".menu");

menu.addEventListener("click", event => {
    const toggle = event.target.closest(".menu__toggle");

    if (!toggle) {
        return;
    }

    toggle.nextElementSibling?.classList.toggle("hidden");
});
```

Тут використано:

- DOM;
- спливання;
- делегування;
- `closest`;
- зміну CSS-класу;
- один обробник для контейнера.

## 30. Типові помилки

### Втрата `this`

```javascript
const method = object.method;
method();
```

Рішення: правильний виклик, `bind` або обгортка.

### Неправильна стрілка

Стрілка не має власного `this` і не є конструктором.

### Методи в кожному екземплярі

Якщо не потрібне приватне замикання, спільний метод краще розмістити у прототипі або класі.

### Плутанина прототипів

Виклик базового конструктора не встановлює успадкування методів.

### Глобальні залежності

Залежності слід імпортувати явно.

### Анонімний обробник

Його не можна зняти без збереженого посилання.

## 31. Підсумки

JavaScript поєднує такі механізми:

- лексична область визначає доступ до змінних;
- замикання зберігає лексичне оточення;
- `this` залежить від способу виклику;
- стрілка лексично захоплює `this`;
- успадкування працює через прототипний ланцюжок;
- класи спрощують запис прототипної моделі;
- модулі ізолюють реалізацію й роблять залежності явними;
- спливання дозволяє делегувати події.

Основна логіка:

```text
область видимості
→ замикання
→ контекст виклику
→ прототипи
→ класи
→ модулі
→ структурований вебзастосунок
```
