# Як подати й продемонструвати матеріал L6 (HTTP)

Окремий сценарій для викладача: порядок показів, команди запуску, що сказати біля екрана. Деталі файлів — у [`README.md`](README.md).

**Час на живі демо:** орієнтовно 20–25 хв (блок «DevTools, curl і простий сервер» з плану).

**Обов’язковий мінімум на парі:** кроки A → B → C.  
**Якщо є час:** D → E.  
**Опційно / окремий семінар:** F (Servlet).

Перед парою переконайтесь, що в PATH є `python` і `curl` (або Git Bash / WSL).

## Підготовка (1–2 хв до початку)

1. Відкрити два термінали й браузер (DevTools → Network).
2. За бажанням заздалегідь створити venv для Flask:

```bash
cd "Lections/L6 - HTTP/src/2.flask_routes"
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

3. Не тримати зайві процеси на портах **8001**, **8002**, **8080**.

## A. Network у браузері (3–5 хв)

**Мета:** показати «сирі» HTTP-повідомлення без власного коду.

1. Відкрити будь-яку багатофайлову сторінку курсу (наприклад HTML+CSS з L2/L3) через локальний сервер або CDN-приклад.
2. F12 → Network → Reload.
3. Показати окремі рядки: HTML, CSS, JS, картинка.
4. Клікнути один запит → Headers: method, status, `Host`, `Content-Type`, `Content-Length`.

**Що акцентувати**

- клієнт ініціює кожен запит;
- одна вкладка = багато пар request/response;
- `Content-Type` визначає, як браузер інтерпретує тіло.

## B. Найпростіший сервер (7–8 хв) — головне демо

**Каталог:** `src/1.simple_http_server/`  
**Порт:** 8001

### Запуск

```bash
cd "Lections/L6 - HTTP/src/1.simple_http_server"
python http_simple_server.py
```

Очікуваний вивід: `Serving at port 8001`.

### Показ 1 — динамічна відповідь

1. Браузер: http://localhost:8001/hello  
2. Паралельно: `curl -v http://localhost:8001/hello`

**Що показати на екрані**

- у терміналі сервера рядок `self.path=...`;
- у `curl -v`: стартовий рядок запиту, заголовки, `HTTP/1.0 200 OK` (або 1.1), тіло;
- у браузері тіло виглядає як текст із «тегами» — бо `Content-Type: text/plain`.

**Фраза для аудиторії:** «Сервер сам вирішує код стану, заголовки й тіло. Браузер лише відображає те, що йому сказали заголовки».

### Показ 2 — статика з диска

1. http://localhost:8001/ → `index.html`
2. http://localhost:8001/немає-такого → 404

**Що акцентувати:** той самий сервер і той самий протокол; відрізняється лише логіка формування відповіді (код vs файл).

### Показ 3 (опційно) — змінити Content-Type live

У `http_simple_server.py` тимчасово замінити:

```python
self.send_header("Content-type", "text/html")
```

Перезапустити сервер → `/hello` відрендериться як HTML. Повернути `text/plain` після демо.

## C. curl як клієнт командного рядка (3–4 хв)

Сервер з кроку B уже працює.

```bash
curl -v http://localhost:8001/hello
curl -v -H "User-Agent: Classroom-Demo" http://localhost:8001/hello
curl -I http://localhost:8001/
```

**Що акцентувати**

- `curl` — HTTP-клієнт без GUI;
- `-v` показує те, що студенти потім побачать у Network;
- `-H` додає власний заголовок;
- `-I` ≈ метод `HEAD` (лише заголовки).

Після цього блоку сервер на 8001 можна зупинити (Ctrl+C), якщо переходите до Flask.

## D. Flask: параметри й заголовки (5–6 хв)

**Каталог:** `src/2.flask_routes/`  
**Порт:** 8002

### Запуск

```bash
cd "Lections/L6 - HTTP/src/2.flask_routes"
.venv\Scripts\activate
python flask_based_server.py
```

### Покази

```bash
curl -v "http://localhost:8002/hello?name=Alice"
curl -v -H "X-token: secret" "http://localhost:8002/hello2?name=Bob"
curl -v -X POST -d "name=Carol" http://localhost:8002/hello2
```

Браузер: http://localhost:8002/hello?name=Student

**Що акцентувати**

- query (`?name=`) vs тіло форми (`POST` + `-d`);
- сервер читає `User-Agent` і власний `X-token`;
- фреймворк приховує сокети, але HTTP-повідомлення ті самі.

## E. Клієнт без браузера (3–4 хв)

Залишити Flask (8002) запущеним.

```bash
cd "Lections/L6 - HTTP/src/4.http_client_python"
python http_client.py
```

**Що акцентувати:** Python-скрипт — теж HTTP-клієнт; відповідь друкується в консоль, а не в DOM.

Якщо показуєте Java — спочатку знову `1.simple_http_server` (8001), потім `HttpClient` з `5.http_client_java`. Коротко: «той самий GET, інша мова».

Notebook `urllib.ipynb` — запасний варіант для лабораторної / самостійної роботи, не обов’язковий на лекції.


**Що акцентувати**

На основній лекції HTTP достатньо одного `curl` з JSON у терміналі.

## Типові збої

| Симптом | Що перевірити |
|---------|----------------|
| `Address already in use` | Звільнити порт або змінити `PORT` у скрипті |
| Браузер «не відкриває» `/hello` | Сервер запущено з правильного каталогу? Дивитись вивід терміналу |
| Flask: `ModuleNotFoundError: flask` | Активовано venv? `pip install -r requirements.txt` |
| Клієнт: `Connection refused` | Сервер відповідного порту не запущений |
| Java client порожня відповідь / помилка | Чи працює `http://localhost:8001/hello` у браузері |

Зупинка серверів: Ctrl+C у відповідному терміналі.
