# Лекція 4. Вступ до JavaScript

## 1. Місце JavaScript у веброзробці

Клієнтська частина вебзастосунку базується на трьох основних технологіях:

- **HTML** описує структуру й семантику;
- **CSS** визначає оформлення;
- **JavaScript** реалізує поведінку й обчислення.

JavaScript дозволяє реагувати на дії користувача, перевіряти введення, змінювати DOM, завантажувати дані із сервера й оновлювати частини сторінки без повного перезавантаження.

Сьогодні JavaScript використовується не лише в браузері. Node.js дозволяє запускати його на сервері, у консольних інструментах і допоміжних програмах. Тому мова давно вийшла за межі простих браузерних сценаріїв.

## 2. JavaScript та ECMAScript

**ECMAScript** — стандарт, який визначає синтаксис, типи даних, оператори, функції, об’єкти та вбудовані конструкції мови.

JavaScript — поширена реалізація цього стандарту в певному середовищі.

DOM, `window`, `document`, `fetch` і таймери не є частиною самого ECMAScript. Їх надає браузер. Node.js має інший набір API: роботу з файлами, процесами й мережею.

JavaScript не є різновидом Java. Це окремі, зовсім не пов'язані між собою, мови програмування.

TypeScript додає статичну перевірку типів і компілюється у JavaScript. Він допомагає раніше знаходити помилки у великих проєктах.

## 3. Підключення сценарію

Вбудований код:

```html
<script>
    console.log("Сторінку завантажено");
</script>
```

Зовнішній файл:

```html
<script src="js/main.js" defer></script>
```

`defer` завантажує файл паралельно з HTML, але виконує його після побудови документа. Це зручно для коду, який звертається до DOM.

`async` також завантажує файл паралельно, але виконує одразу після готовності, тому порядок кількох сценаріїв не гарантується.

Рекомендована структура:

```text
index.html — структура
style.css  — оформлення
main.js    — поведінка
```

## 4. Змінні

### `let`

```javascript
let counter = 0;
counter += 1;
```

Має блокову область видимості.

### `const`

```javascript
const courseName = "Основи вебтехнологій";
```

Забороняє повторне присвоєння. Для об’єкта фіксується посилання, але сам об’єкт можна змінювати:

```javascript
const user = { name: "Anna" };
user.name = "Olena";
```

### `var`

```javascript
var oldValue = 10;
```

`var` має функціональну область видимості й історичні особливості підняття оголошень (hoisting).

```javascript
if (true) {
    var value = 10;
}

console.log(value); // 10
```

У сучасному коді переважно використовують `const`, а за потреби повторного присвоєння — `let`.

## 5. Типи даних

```javascript
const count = 10;          // number
const title = "Lecture";   // string
const enabled = true;      // boolean
let result;                // undefined
const empty = null;        // null
const large = 123n;        // bigint
const key = Symbol("id");  // symbol
const user = { name: "A" };// object
```

Перевірка:

```javascript
typeof count; // "number"
```

Історична особливість:

```javascript
typeof null; // "object"
```

## 6. Динамічна й слабка типізація

Тип належить значенню, а не змінній:

```javascript
let value = 10;
value = "ten";
```

JavaScript часто виконує неявне перетворення:

```javascript
"5" + 2; // "52"
"5" - 2; // 3
```

Краще за можливості робити перетворення явно:

```javascript
Number("42");
String(42);
Boolean(1);
```

Невдале числове перетворення дає `NaN`:

```javascript
Number("abc"); // NaN
Number.isNaN(Number("abc")); // true
```

## 7. Truthy і falsy

Хибними в умовах є:

- `false`;
- `0`;
- `""`;
- `null`;
- `undefined`;
- `NaN`.

Порожній масив і порожній об’єкт є істинними.

```javascript
if ([]) {
    console.log("Масив є truthy");
}
```

## 8. Порівняння

Нестрога рівність виконує приведення типів:

```javascript
5 == "5"; // true
```

Строга рівність порівнює тип і значення:

```javascript
5 === "5"; // false
```

Для передбачуваної поведінки слід використовувати `===` і `!==`.

## 9. Умови й цикли

```javascript
if (score >= 90) {
    console.log("Відмінно");
} else if (score >= 60) {
    console.log("Зараховано");
} else {
    console.log("Не зараховано");
}
```

Тернарний оператор:

```javascript
const status = score >= 60 ? "passed" : "failed";
```

Цикл:

```javascript
for (let i = 0; i < 5; i += 1) {
    console.log(i);
}
```

## 10. Функції

Оголошення:

```javascript
function add(a, b) {
    return a + b;
}
```

Функціональний вираз:

```javascript
const multiply = function (a, b) {
    return a * b;
};
```

Стрілочна функція:

```javascript
const square = value => value * value;
```

Функції є значеннями:

```javascript
function execute(operation, a, b) {
    return operation(a, b);
}

execute(add, 2, 3);
```

Кількість аргументів не перевіряється суворо:

```javascript
function show(a, b) {
    console.log(a, b);
}

show(1);       // b === undefined
show(1, 2, 3); // зайвий аргумент не використано
```

## 11. Масиви

```javascript
const technologies = ["HTML", "CSS", "JavaScript"];
```

Масиви JavaScript:

- мають динамічну довжину;
- можуть містити різні типи;
- є об’єктами;
- допускають пропуски індексів.

```javascript
technologies.push("TypeScript");
technologies.pop();
```

Перебір:

```javascript
for (const item of technologies) {
    console.log(item);
}
```

Методи вищого рівня:

```javascript
const lengths = technologies.map(item => item.length);
const filtered = technologies.filter(item => item.length > 3);
technologies.forEach(item => console.log(item));
```

## 12. Об’єкти

```javascript
const student = {
    name: "Ivan",
    year: 2,
    showInfo() {
        console.log(`${this.name}, курс ${this.year}`);
    }
};
```

Доступ:

```javascript
student.name;
student["year"];
```

Динамічна зміна:

```javascript
student.group = "KN-101";
delete student.year;
```

Об’єкти передаються через посилання:

```javascript
const first = { value: 1 };
const second = first;
second.value = 10;

console.log(first.value); // 10
```

## 13. JSON

JSON — текстовий формат обміну даними.

```json
{
  "name": "Ivan",
  "year": 2
}
```

JSON суворіший за літерал JavaScript:

- ключі й рядки — у подвійних лапках;
- немає функцій;
- немає `undefined`;
- не допускається кома після останнього елемента.

```javascript
const text = JSON.stringify(student);
const restored = JSON.parse(text);
```

## 14. DOM

Браузер перетворює HTML на DOM-дерево. JavaScript може змінювати його вузли.

```javascript
const title = document.querySelector("h1");
title.textContent = "Оновлений заголовок";
```

Пошук:

```javascript
document.getElementById("main");
document.querySelector(".button");
document.querySelectorAll("nav a");
```

Класи:

```javascript
element.classList.add("active");
element.classList.remove("hidden");
element.classList.toggle("selected");
```

Створення:

```javascript
const item = document.createElement("li");
item.textContent = "Новий пункт";
document.querySelector("ul").append(item);
```

## 15. Події

Сучасний спосіб:

```javascript
const button = document.querySelector("#save-button");

button.addEventListener("click", event => {
    console.log("Натиснуто", event);
});
```

HTML-атрибут:

```html
<button onclick="save()">Зберегти</button>
```

працює, але змішує поведінку з розміткою. `addEventListener` дозволяє додати кілька обробників і надалі їх видалити.

## 16. Однопотокова модель

Основний JavaScript-код виконується в одному потоці. Тривала функція блокує інтерфейс.

Таймери й мережеві операції організовано асинхронно. Після завершення їхні callback потрапляють у чергу.

```javascript
console.log("A");

setTimeout(() => console.log("B"), 0);

console.log("C");
```

Результат:

```text
A
C
B
```

Таймер не перериває поточний код.

## 17. Налагодження

DevTools дозволяє:

- читати повідомлення Console;
- встановлювати breakpoint;
- виконувати код покроково;
- переглядати локальні змінні;
- аналізувати стек викликів;
- зупинятися на винятках.

Послідовність:

```text
прочитати помилку
→ перейти до файла і рядка
→ поставити breakpoint
→ перевірити значення
→ виправити логіку
```

## 18. Підсумки

JavaScript є динамічною мовою з гнучкими типами, функціями як значеннями, динамічними масивами й об’єктами.

Для надійного коду важливо:

- використовувати `const` і `let`;
- надавати перевагу `===`;
- явно перетворювати типи;
- відокремлювати JavaScript від HTML;
- досліджувати код через DevTools;
- розуміти різницю між ECMAScript і браузерними API.

Ці поняття є основою для наступної лекції про замикання, `this`, прототипи, класи та модулі.
