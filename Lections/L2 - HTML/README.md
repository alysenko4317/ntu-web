# Лекція 2. HTML та основи клієнтської веб-розробки

## Тема

**Мова розмітки HTML, структура веб-сторінки та базові засоби клієнтської веб-розробки.**

## Основний напрямок лекції

Лекція присвячена HTML як основі структури вебсторінок і клієнтської частини вебзастосунків. Розглядаються будова HTML-документа, теги, атрибути, текстові елементи, посилання, списки, таблиці, зображення, форми та семантична розмітка. Пояснюється розподіл ролей між HTML, CSS і JavaScript.

Окрему увагу приділено сучасним браузерним можливостям, локальному запуску сторінок через вебсервер, а також організації середовища розробки за допомогою WSL і Docker. На практичних прикладах демонструється розгортання статичних і динамічних вебзастосунків та взаємодія компонентів у контейнеризованому середовищі.

## Розглянуті питання

- призначення та історія розвитку HTML;
- HTML як мова розмітки, а не мова програмування;
- структура HTML-документа;
- теги, атрибути та вкладені елементи;
- текст, посилання, списки, таблиці та зображення;
- форми й елементи введення даних;
- розподіл ролей між HTML, CSS і JavaScript;
- семантична структура вебсторінки;
- сучасні можливості браузерних вебзастосунків;
- кешування, Service Worker та офлайн-робота;
- локальний вебсервер і тестування сторінок;
- використання WSL у веброзробці;
- основи Docker і контейнеризації;
- створення та запуск вебконтейнерів;
- багатокомпонентні застосунки й Docker Compose.

## Демонстраційні приклади

Усі приклади розташовані в каталозі [`src/`](src/) і згруповані за тематикою.

### Основи HTML — [`src/basics/`](src/basics/)

Прості HTML-файли, що ілюструють базові елементи розмітки. Побудовані за принципом поступового ускладнення: від мінімальної сторінки до семантичної структури з підключенням зовнішніх бібліотек.

| Файл | Тема | Що демонструє |
|------|------|---------------|
| [`1.hello.html`](src/basics/1.hello.html) | Мінімальна HTML-сторінка | Базова структура документа (`<!DOCTYPE>`, `<html>`, `<head>`, `<body>`), заголовки, абзаци, горизонтальна лінія, ненумерований список, гіперпосилання |
| [`2.table.html`](src/basics/2.table.html) | Таблиці | Елементи `<table>`, `<thead>`, `<tbody>`, `<caption>`, об'єднання комірок (`rowspan`, `colspan`) |
| [`3.form.html`](src/basics/3.form.html) | Форма без стилів | Тег `<form>`, елементи введення (`<input>`, `<label>`), атрибути `type`, `required`, `method` |
| [`4.form_styled.html`](src/basics/4.form_styled.html) | Форма з Bootstrap | Та сама форма з підключенням CSS-фреймворку Bootstrap через CDN — демонстрація розділення структури (HTML) та оформлення (CSS) |
| [`5.semantic.html`](src/basics/5.semantic.html) | Семантична структура HTML5 | Семантичні теги `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>`; вбудовані стилі для наочності |

**Як переглянути.** Відкрити безпосередньо у браузері (подвійний клік по файлу або `Ctrl+O`). Для `4.form_styled.html` потрібне підключення до інтернету (Bootstrap CDN).

> **Зверніть увагу:** `1.hello.html` містить навмисні неточності (дублювання `<p>`, незакриті `<li>`), щоб продемонструвати, як браузер «прощає» помилки в розмітці. Порівняйте вихідний код із тим, що показує інспектор елементів (F12 → Elements) — браузер самостійно додасть відсутні теги.

### Docker — [`src/docker/`](src/docker/)

Знайомство з Docker як інструментом для розгортання веб-застосунків. Три приклади побудовані за принципом поступового ускладнення: від статичного HTML до динамічних серверних застосунків.

| Каталог | Що всередині | Що демонструє |
|---------|-------------|---------------|
| [`1.docker-apache-vanilla/`](src/docker/1.docker-apache-vanilla/) | `Dockerfile`, `docker-compose.yml`, статичний `index.html` | Мінімальний Apache-контейнер; `docker build`, `docker run`, `docker-compose up`; volume-монтування для live-редагування |
| [`2.docker-apache-php/`](src/docker/2.docker-apache-php/) | `Dockerfile`, PHP-сторінки (`index.php`, `greet.php`, `info.php`) | Офіційний образ `php:8.1-apache`; серверний рендеринг, обробка GET-параметрів, `phpinfo()` |
| [`3.docker-apache-python/`](src/docker/3.docker-apache-python/) | `Dockerfile`, Flask-застосунок, `mod_wsgi` конфігурація | Apache + Python: встановлення пакетів у контейнері, WSGI-інтеграція, обслуговування статики й динаміки одним сервером |

**Як запустити.** Кожен приклад має `build.bat` і `run.bat`:

```
cd "Lections/L2 - HTML/src/docker/1.docker-apache-vanilla"
build.bat          # docker build -t my-apache2 .
run.bat            # docker run -dit -p 8081:80 my-apache2
```

Після запуску сторінка доступна за адресою **http://localhost:8081**. Для `1.docker-apache-vanilla` також є `docker-compose.yml` з монтуванням каталогу (порт 8082) — зміни у файлах відразу відображаються в браузері без перезбірки образу.

> **Порівняйте три варіанти:** однакова задача (веб-сервер у Docker) вирішується різними стеками — чистий HTML, PHP, Python/Flask. Dockerfile поступово ускладнюється від 2 рядків (vanilla) до повноцінної збірки з установкою пакетів (python).

### Сучасні веб-API — [`src/js/`](src/js/)

Демонстрація можливостей сучасних браузерних API, які перетворюють веб-сторінку на повноцінний застосунок. Приклади ілюструють теми лекції про Canvas, WebGL і Web Workers.

| Каталог | Тема | Що демонструє |
|---------|------|---------------|
| [`1.canvas/`](src/js/1.canvas/) | HTML5 Canvas та SVG | `canvas.html` — анімація стрибучої кульки через `<canvas>` і `requestAnimationFrame`; `svg.html` — SVG-анімація пульсуючого кола через `<animate>` |
| [`2.webGL/`](src/js/2.webGL/) | WebGL | Ініціалізація WebGL-контексту, вершинний та фрагментний шейдери, рендеринг примітивів (3 варіанти) |
| [`3.webWorker/`](src/js/3.webWorker/) | Web Workers | Виконання важких обчислень (сума 1..N) у фоновому потоці; обмін повідомленнями між Worker і основним скриптом |
| [`4.webWorker.docker-apache/`](src/js/4.webWorker.docker-apache/) | Web Workers у Docker | Той самий приклад Web Worker, розгорнутий через Apache у Docker-контейнері (потрібен для коректної роботи `Worker()` API) |

**Як переглянути.** `1.canvas` і `2.webGL` можна відкрити безпосередньо у браузері. Для `3.webWorker` потрібен веб-сервер (Web Workers не працюють через `file://`). Найпростіший спосіб — `4.webWorker.docker-apache` з Docker:

```
cd "Lections/L2 - HTML/src/js/4.webWorker.docker-apache"
build.bat && run.bat
```

Або будь-який локальний сервер (наприклад, `python -m http.server 8080` у каталозі `3.webWorker/`).

### Що спробувати після лекції

1. Відкрити [`1.hello.html`](src/basics/1.hello.html) у браузері та через DevTools (F12) подивитись, як браузер інтерпретував незакриті теги.
2. Змінити поля у [`3.form.html`](src/basics/3.form.html) — додати `<select>`, `<textarea>`, `<input type="date">` — і перевірити, як форма відправляє дані (зверніть увагу на рядок адреси після натискання «Відправити»).
3. Порівняти [`3.form.html`](src/basics/3.form.html) та [`4.form_styled.html`](src/basics/4.form_styled.html) — зрозуміти, що HTML залишився практично тим самим, а зовнішній вигляд змінився лише завдяки CSS.
4. Вивчити [`5.semantic.html`](src/basics/5.semantic.html) — звернути увагу, як семантичні теги замінюють безлику конструкцію `<div>`.
5. Переглянути еталон ЛР1 ([`Labs/LR1_html_css_basics/`](../../Labs/LR1_html_css_basics/)) як приклад повноцінного багатосторінкового сайту, побудованого на основі тем цієї лекції.
