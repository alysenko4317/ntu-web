# Лекція 2. HTML та розгортання вебсайту в локальному середовищі

## Тема лекції

**HTML як основа структури вебсторінки. Створення, дослідження та запуск статичного вебсайту в локальному середовищі.**

## Мета лекції

Сформувати у студентів розуміння ролі HTML у веброзробці, будови HTML-документа, принципів роботи браузера з розміткою та базових способів локального розгортання вебсайту через вебсервер.

## Очікувані результати навчання

Після лекції студент повинен уміти:

- пояснити призначення HTML та відрізнити мову розмітки від мови програмування;
- описати типову структуру HTML-документа;
- використовувати теги, атрибути, вкладені елементи та спеціальні символи;
- розрізняти блокові, рядкові та семантичні елементи;
- пояснити, як браузер будує DOM-дерево;
- створювати посилання, списки, зображення, таблиці та прості HTML-форми;
- пояснити різницю між методами `GET` і `POST`;
- користуватися основними вкладками інструментів розробника браузера;
- пояснити різницю між відкриттям сторінки через `file://` та її отриманням через HTTP;
- запустити статичний сайт за допомогою локального вебсервера;
- пояснити призначення Apache, WSL і Docker у локальній веброзробці.

## План лекції

### 1. Місце HTML у стеку вебтехнологій

1. Клієнтська та серверна частини вебзастосунку.
2. Ролі HTML, CSS і JavaScript.
3. HTML як мова розмітки, а не мова програмування.
4. Історичний розвиток HTML і перехід до HTML5.
5. Сучасний живий стандарт HTML.

### 2. HTML-документ як текстовий файл

1. Розширення `.html`.
2. Відкриття документа у текстовому редакторі та браузері.
3. Інтерпретація розмітки браузером.
4. Поблажливість браузерів до синтаксичних помилок.
5. Чому не слід покладатися на автоматичне виправлення помилок.

### 3. Базова структура HTML-документа

1. Декларація `<!DOCTYPE html>`.
2. Кореневий елемент `<html>`.
3. Службова частина `<head>`.
4. Видима частина `<body>`.
5. Заголовок документа `<title>`.
6. Метадані, кодування та `<meta charset="UTF-8">`.
7. Підключення стилів і сценаріїв.

### 4. Теги, елементи та атрибути

1. Відкривальні й закривальні теги.
2. Парні та порожні елементи.
3. Вкладеність і деревоподібна структура.
4. Атрибути та їх значення.
5. Глобальні атрибути `id`, `class`, `title`, `hidden`.
6. Коректне закриття та вкладення елементів.

### 5. Спеціальні символи та HTML-сутності

1. Службове значення символів `<`, `>`, `&`.
2. Представлення спеціальних символів.
3. Приклади: `&lt;`, `&gt;`, `&amp;`, `&nbsp;`.
4. Роль кодування UTF-8.

### 6. Основні категорії HTML-елементів

1. Блокові елементи.
2. Рядкові елементи.
3. Контейнери `<div>` і `<span>`.
4. Заголовки та абзаци.
5. Списки.
6. Посилання та якорі.
7. Зображення.
8. Таблиці.
9. Семантичні елементи HTML5: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`.

### 7. DOM-дерево та завантаження ресурсів

1. Перетворення HTML-коду на DOM.
2. Батьківські та дочірні вузли.
3. Використання DOM у JavaScript.
4. Підключення CSS і JavaScript.
5. Вплив сценаріїв на процес побудови сторінки.
6. Розміщення сценаріїв і атрибути `defer` та `async`.

### 8. HTML-форми та передавання даних

1. Призначення елемента `<form>`.
2. Поля `<input>`, `<textarea>`, `<select>` і кнопки.
3. Атрибути `name`, `value`, `type`, `required`.
4. Адреса обробника `action`.
5. Методи `GET` і `POST`.
6. Параметри URL та URL-кодування.
7. Приховані поля.
8. Відсутність стану в HTTP і необхідність передавання контексту.

### 9. Інструменти розробника браузера

1. Відкриття DevTools.
2. Вкладка Elements/Inspector.
3. Перегляд вкладеності елементів.
4. Тимчасове редагування HTML і CSS.
5. Box Model.
6. Console для повідомлень і помилок JavaScript.
7. Network для аналізу HTTP-запитів і відповідей.
8. Перегляд заголовків, кодів стану та параметрів форм.

### 10. Відкриття локального HTML-файлу

1. Запуск сторінки подвійним клацанням.
2. Схема `file://`.
3. Переваги для найпростіших статичних сторінок.
4. Обмеження політики безпеки браузера.
5. Відмінність локального файла від ресурсу, отриманого через HTTP.
6. Випадки, коли потрібен локальний вебсервер.

### 11. Локальний вебсервер

1. Призначення вебсервера.
2. Адреси `localhost` і `127.0.0.1`.
3. Поняття порту.
4. Коренева директорія сайту.
5. Файл `index.html`.
6. Отримання сторінки через `http://localhost`.
7. Прості варіанти локального сервера: Python, Live Server, Apache, Nginx або сервер фреймворку.

### 12. Використання Apache та WSL

1. Встановлення Linux-середовища через WSL.
2. Встановлення Apache.
3. Розміщення файлів сайту в кореневій директорії сервера.
4. Запуск і перевірка служби.
5. Доступ із Windows до сервера у WSL.
6. Особливості IP-адрес, перенаправлення портів і інтеграції WSL.

### 13. Локальне розгортання за допомогою Docker

1. Контейнер як ізольоване середовище виконання.
2. Відмінність контейнера від повноцінної віртуальної машини.
3. Готові образи вебсерверів.
4. Монтування каталогу сайту в контейнер.
5. Відображення порту контейнера на порт хост-системи.
6. Відтворюваність середовища.
7. Docker Compose для багатокомпонентних вебзастосунків.
8. Типові компоненти: вебсервер, PHP або Python, СУБД, кеш і фонові служби.

### 14. Підсумки

1. HTML визначає структуру й семантику вебсторінки.
2. Браузер перетворює HTML на DOM.
3. DevTools є основним засобом дослідження та налагодження сторінки.
4. Відкриття через `file://` придатне лише для простих випадків.
5. Повноцінну локальну розробку слід виконувати через HTTP-сервер.
6. Docker дає змогу автоматизувати та відтворювати середовище розгортання.

## Рекомендовані демонстрації

1. Створити мінімальний файл `index.html` у текстовому редакторі.
2. Відкрити його через `file://` і переглянути адресу.
3. Додати заголовки, абзаци, список, посилання та зображення.
4. Показати DOM-дерево у вкладці Elements.
5. Тимчасово змінити текст і стилі через DevTools.
6. Відправити просту форму методом `GET` і дослідити URL.
7. Переглянути запит у вкладці Network.
8. Запустити каталог через простий локальний сервер.
9. Порівняти адреси `file://...` і `http://localhost:...`.
10. Запустити той самий сайт у контейнері з Apache або Nginx.
