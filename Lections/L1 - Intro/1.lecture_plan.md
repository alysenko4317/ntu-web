# План лекції 1  
## Вступ до вебтехнологій і принципів роботи вебзастосунків

### Тема лекції

Загальне уявлення про вебтехнології, структуру вебзастосунків, клієнт-серверну взаємодію та еволюцію Всесвітньої павутини від статичних вебсторінок до сучасних динамічних систем.

### Мета лекції

Сформувати у студентів цілісне початкове уявлення про те, що таке веброзробка, з яких частин складається вебзастосунок, як взаємодіють клієнт і сервер, яку роль відіграють протоколи, адреси ресурсів і формати даних.

### Очікувані результати навчання

Після лекції студент повинен:

- розрізняти Інтернет і World Wide Web;
- пояснювати клієнт-серверну модель;
- розрізняти frontend, backend і full-stack розробку;
- називати основні технології клієнтської та серверної частин;
- пояснювати різницю між статичними й динамічними вебресурсами;
- описувати основні етапи розвитку вебтехнологій;
- пояснювати призначення HTTP, MIME-типів, IP-адрес, DNS-імен і портів;
- розкладати URL на основні складові;
- розуміти, що вебсервер може повертати не лише готові файли, а й динамічно сформовані документи або дані.

## Орієнтовна структура лекції

### 1. Організаційний вступ і місце дисципліни в підготовці фахівця

1. Мета курсу «Основи вебтехнологій».
2. Необхідний початковий рівень:
   - базові навички програмування;
   - загальне уявлення про файли, мережі та операційні системи.
3. Характер курсу:
   - огляд клієнтської та серверної частин;
   - акцент на технічних принципах, а не лише на дизайні;
   - використання Python або іншої знайомої студентам мови для серверних прикладів.
4. Вебдизайн як окрема сфера діяльності.
5. Роль сучасних інструментів і систем штучного інтелекту в розробці.

### 2. Що таке веброзробка

1. Вебтехнології як набір засобів для створення систем, що взаємодіють через мережу.
2. Основні частини вебзастосунку:
   - клієнтська частина;
   - серверна частина;
   - сховище даних та зовнішні сервіси.
3. Поняття frontend, backend і full-stack.
4. Веббраузер як типовий, але не єдиний клієнт.
5. Приклади інших клієнтів:
   - мобільний застосунок;
   - настільна програма;
   - Telegram-бот;
   - програмна бібліотека;
   - інший сервер.

### 3. Базові клієнтські технології

1. HTML як мова розмітки.
2. CSS як засіб опису зовнішнього вигляду.
3. JavaScript як мова програмування у браузері.
4. Межі між структурою, оформленням і поведінкою сторінки.
5. Вбудовування вебінтерфейсів у настільні та мобільні програми.

### 4. Серверна частина вебзастосунку

1. Призначення backend:
   - обробка запитів;
   - реалізація бізнес-логіки;
   - робота з базами даних;
   - взаємодія із зовнішніми сервісами;
   - формування відповіді клієнту.
2. Мови серверної розробки:
   - Python;
   - PHP;
   - Java;
   - C#;
   - JavaScript/Node.js;
   - інші мови.
3. Вебсервер, прикладна програма та база даних як окремі компоненти.
4. Монолітна й мікросервісна архітектура на оглядовому рівні.

### 5. Інтернет і World Wide Web

1. Інтернет як глобальна мережа взаємопов’язаних мереж.
2. World Wide Web як один із сервісів Інтернету.
3. Інші мережеві сервіси:
   - електронна пошта;
   - передавання файлів;
   - віддалений доступ;
   - обмін повідомленнями.
4. Вебсторінка, вебсайт і вебзастосунок.
5. Централізоване розміщення та оновлення інформації.

### 6. Історичний розвиток вебтехнологій

1. Статичні HTML-документи.
2. Обмеження статичних сторінок.
3. Поява серверних сценаріїв і CGI.
4. Розвиток серверних мов і платформ:
   - Perl;
   - PHP;
   - Java Servlets;
   - ASP/ASP.NET;
   - Python-фреймворки;
   - Ruby on Rails.
5. Поява JavaScript у браузері.
6. Java-аплети як історичний етап.
7. AJAX і асинхронне оновлення частини сторінки.
8. jQuery як популярна бібліотека попереднього покоління.
9. Сучасні клієнтські фреймворки й бібліотеки:
   - React;
   - Angular;
   - інші.
10. Перехід від генерації готового HTML на сервері до API та передавання структурованих даних.
11. Односторінкові застосунки та побудова інтерфейсу на боці клієнта.

### 7. Клієнт-серверна взаємодія

1. Запит клієнта та відповідь сервера.
2. Протокол як узгоджений набір правил.
3. HTTP як прикладний протокол вебу.
4. Можливі типи відповіді:
   - HTML-документ;
   - текст;
   - зображення;
   - таблиця стилів;
   - JavaScript;
   - JSON або інші структуровані дані;
   - файл довільного типу.
5. Статична й динамічна відповідь.
6. Серверний шаблон і підстановка даних.
7. API як програмний інтерфейс між клієнтом і сервером.

### 8. MIME-типи та інтерпретація ресурсів браузером

1. Чому розширення файлу не є достатнім для браузера.
2. HTTP-заголовок `Content-Type`.
3. Приклади MIME-типів:
   - `text/html`;
   - `text/plain`;
   - `text/css`;
   - `application/javascript`;
   - `application/json`;
   - `image/png`.
4. Залежність обробки отриманих даних від їхнього типу.
5. Перегляд запитів і відповідей у Developer Tools.

### 9. Будова URL

1. Загальна схема URL:

   ```text
   scheme://host:port/path?query#fragment
   ```

2. Складові:
   - схема або протокол;
   - ім’я вузла чи IP-адреса;
   - порт;
   - шлях до ресурсу;
   - параметри запиту;
   - фрагмент документа.
3. DNS як механізм перетворення доменного імені на IP-адресу.
4. Порт як ідентифікатор мережевої служби на комп’ютері.
5. Типові порти:
   - HTTP — 80;
   - HTTPS — 443.
6. Фізичні та віртуальні шляхи.
7. Маршрутизація запитів на сучасному сервері.

### 10. Пов’язані ресурси вебсторінки

1. HTML-документ як початкова точка завантаження.
2. Додаткові запити браузера до:
   - CSS-файлів;
   - JavaScript-файлів;
   - зображень;
   - шрифтів;
   - мультимедіа;
   - API.
3. Відносні й абсолютні адреси ресурсів.
4. Посилання на ресурси з HTML, CSS і JavaScript.

### 11. Підсумки лекції

1. Вебзастосунок є розподіленою системою.
2. Браузер — лише один із можливих клієнтів.
3. Сучасний веб виріс зі статичних документів, але перетворився на платформу для повноцінних програм.
4. Основою взаємодії є мережеві протоколи, адресація та узгоджені формати даних.
5. Для подальшого вивчення потрібно окремо розглянути:
   - вебклієнти;
   - вебсервери;
   - HTTP-запити та відповіді;
   - HTML, CSS і JavaScript;
   - серверне програмування;
   - бази даних і API.

## Демонстрації під час лекції

1. Відкрити просту вебсторінку та показати її HTML, CSS і JavaScript.
2. Відкрити Developer Tools → Network.
3. Перезавантажити сторінку й показати:
   - основний HTML-запит;
   - додаткові запити до CSS, JavaScript і зображень;
   - HTTP-статус;
   - заголовок `Content-Type`.
4. Розібрати реальний URL на складові.
5. За допомогою `ping` або `nslookup` продемонструвати перетворення доменного імені на IP-адресу.
6. Показати URL із явно вказаним портом.
7. Порівняти відповідь сервера у вигляді HTML і JSON.
