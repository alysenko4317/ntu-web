# Лабораторна робота №2 — Мережеві утиліти та протоколи

**Мета:** ознайомитися з базовими службовими утилітами для діагностики мережі, вивчити практичні основи роботи мережевих протоколів (ARP, ICMP, DNS, TCP, HTTP, HTTPS).

> У цій лабораторній роботі **не передбачається створення програмного коду**. Результатом є звіт із прикладами виконаних команд, скриншотами та поясненнями.

## Ключові поняття

### IP-адресація

- **IP-адреса** — унікальний ідентифікатор пристрою в мережі (IPv4: `192.168.1.10`, IPv6: `fe80::1`).
- **Маска підмережі** — визначає, яка частина IP-адреси відноситься до мережі, а яка — до хоста (наприклад, `255.255.255.0` або `/24`).
- **Шлюз за замовчуванням** — маршрутизатор, через який пакети виходять за межі локальної мережі.
- **DNS-сервер** — перетворює доменні імена в IP-адреси.

### Основні мережеві протоколи

| Протокол | Рівень | Призначення |
|----------|--------|-------------|
| ARP | Канальний/Мережевий | Відповідність IP → MAC у локальній мережі |
| ICMP | Мережевий | Діагностика (ping, tracert) |
| DNS | Прикладний | Перетворення доменних імен → IP |
| TCP | Транспортний | Надійна передача даних із встановленням з'єднання |
| HTTP | Прикладний | Протокол передачі гіпертексту (порт 80) |
| HTTPS | Прикладний | HTTP із шифруванням TLS (порт 443) |

### Основні TCP/UDP-порти

| Порт | Служба | Протокол |
|------|--------|----------|
| 22 | SSH | TCP |
| 53 | DNS | TCP/UDP |
| 80 | HTTP | TCP |
| 443 | HTTPS | TCP |
| 3306 | MySQL | TCP |
| 5432 | PostgreSQL | TCP |

## Короткий довідник команд

### Перегляд мережевих налаштувань

```bash
# Windows
ipconfig /all

# Linux / WSL
ip addr show
```

### ARP-таблиця

```bash
# Windows
arp -a

# Linux / WSL
ip neigh show
```

### Перевірка доступності вузла

```bash
# Відправити 4 ICMP-запити
ping -n 4 google.com        # Windows
ping -c 4 google.com        # Linux
```

### DNS-запити

```bash
# Визначити IP-адресу за доменним іменем
nslookup example.com

# Розширений запит (Linux / WSL)
dig example.com A
dig example.com MX
```

### Маршрут проходження пакетів

```bash
# Windows
tracert google.com

# Linux / WSL
traceroute google.com
tracepath google.com
```

### Активні з'єднання

```bash
# Windows
netstat -an

# Linux / WSL
ss -tuln
```

### Ручний HTTP-запит (telnet)

```bash
telnet example.com 80
# Після з'єднання ввести:
# GET / HTTP/1.1
# Host: example.com
# (натиснути Enter двічі)
```

### HTTP/HTTPS-запити (curl)

```bash
# Отримати вміст сторінки
curl http://example.com

# Переглянути заголовки відповіді
curl -I https://example.com

# Переглянути повну відповідь із заголовками
curl -v https://example.com
```

### Wi-Fi-профілі (Windows)

```bash
# Список збережених профілів
netsh wlan show profiles

# Деталі профілю (з паролем)
netsh wlan show profile name="NetworkName" key=clear
```

## Порівняння утиліт Windows та Linux

| Задача | Windows | Linux / WSL |
|--------|---------|-------------|
| Мережеві налаштування | `ipconfig /all` | `ip addr show` |
| ARP-таблиця | `arp -a` | `ip neigh show` |
| Доступність вузла | `ping -n 4 host` | `ping -c 4 host` |
| DNS-запит | `nslookup host` | `dig host` / `host host` |
| Маршрут пакетів | `tracert host` | `traceroute host` |
| Активні з'єднання | `netstat -an` | `ss -tuln` |
| Wi-Fi-профілі | `netsh wlan show profiles` | `nmcli con show` |
