# Library Base — PHP-версія (еталон ЛР4)

Демонстраційний PHP-проєкт для лабораторної роботи №4: серверна реєстрація, авторизація, HTTP-сесії та захищений кабінет. Реалізовано на темі **електронної бібліотеки** як референс для викладача та орієнтир для студентів.

## Важливо для студентів

У **ЛР1** ви обираєте власну тематику сайту і розвиваєте її впродовж ЛР3–ЛР8. Цей каталог `src/` — **не шаблон для копіювання «як є»**, а приклад технічної реалізації вимог ЛР4. Виконуйте роботу на базі **свого** сайту з ЛР3; доменні елементи еталону адаптуйте під обрану галузь:

| У демо-еталоні (бібліотека) | У власному проєкті студента |
|-----------------------------|----------------------------|
| `ticket` — номер читацького квитка | власний ідентифікатор (номер клієнта, студентський квиток тощо) |
| `room` — зал бібліотеки | поле за тематикою (категорія, філія, група…) |
| `catalog.php` — каталог книг | каталог товарів, послуг, статей тощо |
| тексти «читач», «бібліотека» | контент обраного домену |

Технічний каркас (PHP, `$_SESSION`, `users.json`, `password_hash()` / `password_verify()`, захист `cabinet.php`) залишається тим самим.

## Що реалізовано

- перехід основних сторінок з `.html` на `.php`;
- спільна динамічна шапка і футер у `includes/layout.php`;
- серверна реєстрація у `register.php`;
- серверний вхід у `login.php` (email або номер читацького квитка);
- вихід із системи у `logout.php`;
- захищений кабінет у `cabinet.php`;
- збереження користувачів у `data/users.json`;
- хешування паролів через `password_hash()`;
- перевірка паролів через `password_verify()`;
- сесія користувача через `$_SESSION['user_email']`;
- `session_regenerate_id(true)` після успішного входу;
- збереження CSS і JavaScript з попередньої статичної версії.

## Як запустити у WSL + Apache

1. Скопіювати каталог `src` у веб-каталог Apache, наприклад:

```bash
sudo cp -r src /var/www/html/library_base_php_lr4
```

2. За потреби надати Apache право запису у каталог `data/`:

```bash
sudo chown -R www-data:www-data /var/www/html/library_base_php_lr4/data
sudo chmod -R 775 /var/www/html/library_base_php_lr4/data
```

3. Відкрити у браузері:

```text
http://localhost/library_base_php_lr4/
```

## Навчальні примітки

Файл `data/users.json` використовується тільки як просте навчальне сховище. У реальному застосунку для користувачів зазвичай використовують базу даних.

Форми `register.php` і `login.php` не мають атрибута `novalidate`, щоб дані надсилалися на сервер для обов'язкової серверної перевірки. Контактна форма (`contacts.php`) залишена з `novalidate` як приклад клієнтської JavaScript-перевірки з ЛР3.

Еталонний код узгоджений із вимогами методички `lab4_php_auth.docx` щодо PHP, сесій, файлового сховища користувачів і захисту кабінету. Тема бібліотеки в еталоні не обмежує студентів у виборі власного домену.

## Індивідуальне завдання (самостійна частина)

Базовий механізм auth (реєстрація, вхід, сесія, кабінет) покрито теорією та еталоном. **Самостійно** студент має реалізувати:

1. **Серверну контактну форму** — `contacts.php`: POST, валідація, збереження в `data/messages.json` (у еталоні форма лише з JS-перевіркою).
2. **Редагування профілю в кабінеті** — `cabinet.php`: зміна імені та доменного поля з оновленням `users.json` (у еталоні кабінет лише для перегляду).

Повний текст завдання, критерії оцінювання та вимоги до звіту — у [`INDIVIDUAL_TASK.md`](INDIVIDUAL_TASK.md) (чернетка для розділу методички).
